## Defect Review — internal/modelspec/

1. **[registry.go, Resolve]** The escape-form branch (`s.Alias == ""`) never validates that Engine/Model are non-empty, so a zero-value `Spec{}` resolves silently to `Resolved{Engine:"", Model:"", Params:{}}` with no error, instead of failing loud like every other entry point in this package — HIGH.

2. **[registry.go, Resolve]** If a malformed `Spec` has both `Alias` and `Engine`/`Model` set, `Resolve` silently does an alias lookup and drops the Engine/Model fields with no error or indication they were ignored — HIGH.

3. **[modelspec.go/registry.go, Spec]** The "exactly one of two shapes" invariant on `Spec` is documented only in comments, not enforced by the type (all fields exported, no constructor/encapsulation) — any caller can build an invalid `Spec` and hand it to `Resolve`, which (per #1/#2) does nothing to defend against it. This is the root design cause of #1 and #2 — HIGH.

4. **[load.go, validateAlias]** An empty-string alias key in `models.yaml` passes `validateCharset` (the range loop over `""` never executes, so no charset failure) and is silently accepted into the registry — but it's permanently unreachable, since `Resolve` treats `Alias == ""` as escape form and never looks it up. A malformed file loads clean and produces a dead entry with no operator feedback — MED/HIGH.

5. **[load.go, validateAlias]** `Defaults` values are checked for known-key and non-empty, but never charset-validated (unlike bracket param values in `parse.go`, which go through `isModelIDChar`). A `models.yaml` default like `effort: "high!"` or one containing spaces loads successfully and flows unvalidated into `Resolved.Params` — an asymmetry with the strict bracket path — MED/HIGH.

6. **[registry.go, Resolve]** `Resolve` never checks that `entry.Defaults` or `s.Params` keys belong to `knownParams`; only `LoadRegistry`-sourced entries and `Parse`-sourced brackets are gated at their respective entry points. A hand-built `Registry`/`Spec` can inject arbitrary param keys into `Resolved.Params` — MED.

7. **[leaf_enforcement_test.go, TestLeafInvariant_AllowlistOnly]** On a `go/parser.ParseFile` failure, the test only `t.Logf`s a warning and continues, silently skipping the leaf-invariant check for that file rather than failing the test — undermines the stated guarantee of catching a stray dependency "with no list maintenance required" for any file broken enough not to parse — MED.

8. **[load.go, LoadRegistry]** Only the first YAML document is decoded; a `models.yaml` containing a second `---`-separated document has that content silently ignored with no error — LOW.

9. **[load.go, LoadRegistry]** An explicit YAML null document (e.g. file content `~` or `null`) decodes successfully with `fileEntries` left `nil` and is silently treated as "no entries" — a distinct code path from the documented io.EOF/empty-file case, not mentioned in the doc comment — LOW.

10. **[parse.go, Parse/parseBracket]** Nested/repeated bracket groups (e.g. `sonnet[a=1][b=2]`, or a value containing `[`/`]`) are rejected, but via a misleading "invalid character in param value" error rather than a clear "malformed bracket" message, since bracket splitting only anchors on the first `[` and last `]` — LOW (error clarity).

11. **[parse.go, validateCharset]** The reported error position `i` is a byte offset from `range`, not a character index; harmless today since the allowed charset is pure ASCII, but could mislead if a non-ASCII rune ever appears after other multi-byte runes — LOW.

12. **[parse.go, grammar]** No escaping mechanism exists for param values, so a value can never contain a comma, `[`, or `]` — an inherent design constraint that isn't documented as a limitation anywhere in the grammar comments — LOW.

13. **[modelspec.go, Registry]** `Registry` is a bare exported map type; nothing prevents a caller from mutating a `Registry` returned by `LoadRegistry` afterward (e.g. `reg["sonnet"] = Entry{}`), silently bypassing all validation `LoadRegistry` performed — LOW/MED.

14. **[modelspec.go, knownParams/knownEngines]** These closed-vocabulary sets are mutable package-level `var` maps rather than something immutable; nothing at the language level stops accidental runtime mutation, which would silently change validation behavior package-wide — LOW.

15. **[registry_test.go, test gap]** No test exercises `Resolve` with a zero-value `Spec{}` or a `Spec` with both `Alias` and `Engine`/`Model` set — the silent-success behavior in #1/#2 is completely untested — MED.

16. **[load_test.go, test gap]** No test covers an empty-string alias key in `models.yaml` (#4) or a `Defaults` value with invalid characters (#5) — MED.

17. **[load_test.go, test gap]** No test covers `LoadRegistry` when the `models.yaml` path is a directory rather than a file (the generic wrapped-error branch is unexercised) — LOW.

18. **[load_test.go, test gap]** No test covers multiple YAML documents or an explicit null-document `models.yaml` (#8/#9) — LOW.

19. **[registry_test.go, TestRegistry_Resolve_NeverMutatesInputs]** Only exercises the case where both registry `Defaults` and spec `Params` are present; the defaults-only (no bracket params) mutation path is untested — LOW.

20. **[template.go/template_test.go]** The "seed-only, never rewritten again" reconcile guarantee described in the package doc comment is enforced entirely by a different package (`configreg`) with zero coverage here — this package only proves the template loads cleanly once, not that it's never re-materialized — LOW (unverified cross-package contract).
