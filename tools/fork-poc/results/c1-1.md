## Findings

1. **[registry.go, Registry.Resolve]** A hand-built `Spec{}` zero value (Alias, Engine, Model all empty) is *not* rejected — since `Alias == ""` it falls into the escape-form branch and returns `Resolved{Engine:"", Model:"", Params:{}}` with a nil error, silently producing a garbage resolution instead of failing loudly. — **HIGH**

2. **[registry.go, Registry.Resolve]** `Resolve` never re-validates the "exactly one shape" invariant documented in modelspec.go: a hand-built `Spec{Alias:"x", Engine:"y", Model:"z"}` (both shapes populated) is accepted, and Engine/Model are silently discarded in favor of the alias lookup with no error or warning. — **MED**

3. **[registry.go, Registry.Resolve]** For escape-form specs, `Resolve` never checks `s.Engine` against `knownEngines` — only `Parse` enforces that. A hand-built `Spec{Engine:"gemini", Model:"x"}` resolves successfully and propagates an unvetted engine downstream. — **MED**

4. **[registry.go, Registry.Resolve / load.go]** `Resolve` trusts `Entry.Engine`/`Entry.Model` unconditionally with no re-validation; a `Registry` constructed by hand (not via `LoadRegistry`) with an empty `Model` or unknown `Engine` resolves silently instead of erroring — validation lives only in `load.go`'s `validateAlias`, which nothing else reuses. — **MED**

5. **[modelspec.go / registry.go]** No exported constructor or `Validate()` method exists for `Spec`/`Entry`/`Registry` to enforce the documented invariants at the type level — since all fields are exported, any caller can build an invariant-violating value that only `Parse`'s callers are protected against. — **MED**

6. **[leaf_enforcement_test.go, TestLeafInvariant_AllowlistOnly]** On a file parse failure, the test does `t.Logf` (warning) and `return nil`, silently skipping the import check for that file rather than failing — a syntactically broken production file could hide a disallowed import and the leaf invariant would report false success. — **MED**

7. **[leaf_enforcement_test.go, isStdlib heuristic]** Classifying "stdlib" as "no dot in the first path segment" is a heuristic that would misclassify any internal/vendored import path lacking a dot (e.g., a bare monorepo alias) as stdlib, silently bypassing the allowlist check. Currently safe only because loomyard's module path happens to contain a dot. — **LOW**

8. **[parse.go, Parse — bracket splitting]** Only a single bracket group is structurally recognized; a second bracket (e.g. `sonnet[a=1][b=2]`) is not rejected with a clear "unexpected extra bracket" error — it's absorbed into the first bracket's inner content and fails later with a confusing "invalid character" message on the param value instead of a precise structural error. — **LOW**

9. **[parse.go, Parse — bracket splitting]** Nested-bracket-like input (e.g. `sonnet[a=[1]]`) is not explicitly rejected as a structural error; it's caught downstream only because `[`/`]` fail the param-value charset check, producing a less specific error than the grammar deserves. — **LOW**

10. **[parse.go, whitespace check]** `unicode.IsSpace` rejects the full Unicode whitespace set, while charset validation only allows ASCII `[a-z0-9._-]` anyway — the whitespace pre-check is redundant with charset validation except for producing a more specific error message; harmless but duplicated logic worth noting. — **LOW**

11. **[load.go, LoadRegistry]** Uses a single `Decode` call, so a multi-document YAML file (`---` separated) silently only processes the first document with no error about the extra documents. — **LOW**

12. **[load.go, LoadRegistry]** No caching — every call re-reads and re-parses `models.yaml` from disk; fine for current usage patterns but a latent perf cost if called in a hot path. — **LOW**

13. **[modelspec.go, knownParams/knownEngines]** These are package-level mutable `var` maps rather than something immutable; nothing in-package prevents accidental mutation at runtime (e.g., a stray `knownParams["x"] = true` in a future change) since Go has no const maps — a self-inflicted risk with no test guarding the vocab's contents stay fixed. — **LOW**

14. **[modelspec.go, Resolved.Params contract]** Consumers must hardcode the magic strings `"effort"`/`"version"` (per the package doc's own example) to read `Resolved.Params` — there are no exported constants for known param keys, so every consumer duplicates these literals with no compiler-checked link back to `knownParams`. — **LOW**

15. **[registry.go, Registry (concurrency)]** `Registry` is a plain `map[string]Entry` with no documented concurrency contract; concurrent `Resolve` reads while another goroutine mutates the map (a legitimate pattern if a caller wants "live-reloadable" registries) would race, and this isn't called out anywhere. — **LOW**

16. **[Test coverage gap]** No test exercises the full pipeline `Parse(s)` → `Registry.Resolve(spec)` together (parse_test.go and registry_test.go each construct `Spec` literals directly) — an integration-level regression between the two stages (e.g., a Parse output shape Resolve doesn't handle) wouldn't be caught. — **MED**

17. **[Test coverage gap]** No test constructs a hand-built invariant-violating `Spec` (both alias and engine/model set, or all-empty) to pin down `Resolve`'s behavior — this gap is precisely why findings #1–#3 exist undetected. — **MED**

18. **[load_test.go, coverage gap]** No test exercises a `models.yaml` with an entry that sets `defaults: {}` (explicit empty map) versus omitted `defaults` (nil) to confirm both are treated identically. — **LOW**

19. **[load_test.go, coverage gap]** No test covers `models.yaml` being a directory instead of a regular file (the "any other read error" path in `LoadRegistry`'s doc comment) — the wrapped-error branch is documented but untested. — **LOW**

20. **[load.go, LoadRegistry — duplicate YAML keys]** Behavior when `models.yaml` contains the same alias key twice (e.g., two `sonnet:` blocks) is unverified by any test; depending on yaml.v3's duplicate-key handling this could silently take the last occurrence rather than erroring, undermining the "loud error" philosophy elsewhere in the package. — **LOW**

21. **[template.yaml / template_test.go]** Nothing enforces that `template.yaml`'s content stays in sync with `knownParams`/`knownEngines`/`builtins()` beyond the one round-trip test — e.g., a future new built-in alias added to `builtins()` wouldn't be caught as "missing from the seed template" by any test. — **LOW**

22. **[parse.go, parseBracket]** Param order is not preserved (map iteration), and duplicate-key detection depends on insertion order into a Go map during a single left-to-right scan — correct today, but the "first duplicate wins the error" behavior isn't explicitly tested for which duplicate index is reported. — **LOW**
