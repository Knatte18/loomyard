## Findings

### [BLOCKING] board.md config-path updates missing from all cards

**Location:** Batch 3 / Cards 16–17
**Issue:** `docs/modules/board.md` is listed in "All Files Touched" and the `discussion.md` technical context table explicitly identifies config-path references at lines 153, 227, 234, 262, and 299 as needing migration to `_lyx/config/board.yaml`. Card 16 edits `board.md` but its Requirements cover only Hub/Prime terminology and a weft pointer — no config-path updates. Card 17 handles config-path doc updates but its Edits list does not include `board.md`. Neither card assigns these five sites.
**Fix:** Add the five board.md config-path rename sites to Card 17's Requirements and Edits (or Card 16's Requirements), mirroring the treatment of `docs/shared-libs/config.md`.

### [NIT] Card 11 misstates concurrency_test.go site count

**Location:** Batch 2 / Card 11
**Issue:** Card 11 says the `seedWiki` function in `concurrency_test.go` "seeds `_lyx/board.yaml` at 3 sites." The `seedWiki` function is defined once in `bench_test.go` with a single path string; `concurrency_test.go` contains three calls to that shared function but no independent path strings. An implementer searching `concurrency_test.go` for three literal path strings will find none.
**Fix:** Reword to: "update the single `board.yaml` path string in `bench_test.go`'s `seedWiki`; all three call sites in `concurrency_test.go` are covered automatically."

### [NIT] `HubName()` → `PrimeName()` not called out in Card 13 Requirements

**Location:** Batch 3 / Card 13
**Issue:** `docs/overview.md` line 47 lists `HubName()` in the Layout methods summary. Card 13 requirement (1) says replace "hub/Hub" references to the main worktree with "Prime" — this should logically cover `HubName()`, but the identifier is not named explicitly, creating a risk of the implementer missing it.
**Fix:** Add an explicit item to Card 13 Requirements: "replace `HubName()` → `PrimeName()` in the Layout methods list (line 47)."
