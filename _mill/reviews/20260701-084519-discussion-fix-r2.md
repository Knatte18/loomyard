# Discussion fix — round 2

Source review: `_mill/reviews/` discussion-review round 2 (verdict APPROVE, 2 NOTE findings).

## Fixed

- **RealDirectoryGuard test setup ordering is implicit** — `seedLyxJunction` (inside
  `WireJunctions`) itself hard-errors on a real non-junction `_lyx` directory, so a
  single-phase "pre-create the real directory, then init" setup is infeasible (init
  would never succeed, so gitignore/exclude/weft-content would never exist to assert
  against). Rewrote `TestRunInit_Undo_RealDirectoryGuard`'s description in the Testing
  section to spell out the required two-phase setup: init succeeds first, then the
  junction is swapped for a real directory, then `--undo` runs.
- **No explicit case for commit-succeeds/push-fails then rerun** — a plausible partial
  state given `weftengine.Push`'s own rebase-retry logic. Expanded
  `TestRunInit_Undo_PartialRecovery`'s scope to also cover this case, and strengthened
  the "Weft-side content" Decision to explicitly state that `weftengine.Push` must be
  called unconditionally after `Commit` succeeds — not gated on `Commit`'s `committed`
  return value — since gating it would strand an already-committed-but-unpushed
  deletion forever on rerun (a real design bug this NOTE surfaced, not just a test-
  coverage gap).

## Pushed Back

None — both findings were factually accurate and low-harm to fix.
