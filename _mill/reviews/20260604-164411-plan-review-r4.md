## Findings

### [BLOCKING] runGit drops exit code; commitPush step 2 unimplementable
**Location:** Batch 5 / Card 11
**Issue:** `runGit` is specified to return `WikiPushError(stderr)` for any non-zero exit, discarding the exit code. `commitPush` step 2 then requires checking whether exit code is exactly 1 (`"if exit code ≠ 1, return error"`) to distinguish "staged changes" from a git error — but that value is gone by the time `runGit` returns.
**Fix:** Either add `ExitCode int` to `WikiPushError` (struct form, already allowed by the "or simple struct types" wording) and populate it in `runGit`, or return the raw exit code as a fourth value from `runGit`, so `commitPush` can test `exitCode == 1`.

### [NIT] task.go in git_test.go Context unused
**Location:** Batch 5 / Card 12
**Issue:** `git_test.go` tests `pathGuard`, `atomicWrite`, `pull`, and `commitPush` — none reference `Task`.
**Fix:** Remove `task.go` from Card 12's Context list.

### [NIT] setPhase []interface{} → *[2]interface{} conversion implicit
**Location:** Batch 6 / Card 16
**Issue:** JSON unmarshal gives `[]interface{}` for the `set_phase` array; `Wiki.MergeTasks` expects `*[2]interface{}`; the required slice-to-array conversion is unmentioned in card 16's Requirements.
**Fix:** Add one sentence specifying the conversion (validate length == 2, copy into `[2]interface{}{arr[0], arr[1]}`).

### [NIT] slug_for_msg fallback for non-upsert write ops unspecified
**Location:** Batch 6 / Card 15
**Issue:** `writeOp` derives the commit message slug for `UpsertTask`/`MergeTasks` but the plan is silent for `SetPhase`, `RemoveTask`, `SetDeps`, and `UpsertTasksBatch`.
**Fix:** Add a sentence naming the fallback string (e.g. `"update"`) for write ops that have no single target slug.
