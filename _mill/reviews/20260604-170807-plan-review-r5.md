## Findings

### [NIT] Sidebar `display_title` undefined in render spec
**Location:** Batch 4 / Card 9
**Issue:** The Home.md section defines `display_title = "**#NNN:** " + title + " [layer]"`. The Sidebar section reuses `<display_title>` without redefining it; an implementer following the literal spec would include the bold `#NNN:` prefix in sidebar bullets, which is wrong.
**Fix:** Explicitly state that Sidebar bullets use `extendedTitle(t, layer)` (already in Context via `layer.go`), not the heading `display_title`.

### [NIT] `render` does not explicitly delegate to `renderOrder`
**Location:** Batch 4 / Card 9
**Issue:** Card 9 re-specifies the full bucket ordering and per-ID sort that `renderOrder` already implements; the implementer may duplicate rather than call the existing helper.
**Fix:** Add one sentence to Card 9 requirements: "Call `renderOrder(tasks)` to obtain a `[]TaskWithLayer` in canonical bucket order; iterate over the result."

### [NIT] `SetPhase` silent no-op vs `SetDeps` error on missing slug
**Location:** Batch 2 / Card 5
**Issue:** `SetPhase` returns nil (no-op) for a missing task; `SetDeps` returns an error for a missing slug — inconsistent for two structurally similar methods.
**Fix:** This appears intentional (test `l` explicitly verifies the nil return), so add a comment in the requirements noting the asymmetry is by design to match Python behavior.
