## Findings

### [NIT] Batch 3 depends-on should list batches 1 and 2
**Location:** `00-overview.md` — batch 3 entry
**Issue:** `depends-on: []` for docs-and-roadmap means the orchestrator may run it before terminology-rename or config-path-migration; the docs then describe `PrimeName()` and `_lyx/config/` while the code still has `HubName()` and flat `_lyx/`.
**Fix:** Set `depends-on: [terminology-rename, config-path-migration]` for batch 3; since `verify: null` this has no build cost and removes the ambiguity.

### [NIT] Card 7 uses undefined variable name `baseDir` in mkdir requirement
**Location:** Batch 2 / Card 7 (`init.go`)
**Issue:** The requirement text says `os.MkdirAll(filepath.Join(baseDir, "_lyx", "config"), 0755)` but `init.go` defines `lyxDir` (not `baseDir`), so the literal snippet would not compile without adaptation.
**Fix:** Express the mkdir as `filepath.Join(lyxDir, "config")` or `filepath.Join(cwd, "_lyx", "config")` to match `init.go`'s actual variable names.
