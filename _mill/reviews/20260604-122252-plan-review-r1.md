## Findings

### [BLOCKING] atomicWrite signature contradicts shared decision
**Location:** 00-overview.md Shared Decisions / Batch 5 Card 11 / Batch 2 Card 4
**Issue:** Overview documents `atomicWrite(dest string, content []byte)` (2-param, `[]byte`); Card 11 specifies `atomicWrite(wikiPath, relPath, content string)` (3-param, `string`); Card 4 calls the 3-param form. An implementer following the shared decision writes the wrong function signature.
**Fix:** Update the `atomic_write` shared decision to show the actual signature: `atomicWrite(wikiPath, relPath string, content string) error`.

### [BLOCKING] Batch 2 verify fails â€” depends-on missing Batch 5
**Location:** 00-overview.md Batch Index / Batch 2 Card 4
**Issue:** `Store.Save()` calls `atomicWrite` which is defined in `git.go` (Batch 5). Batch 2's `depends-on: [1, 3]` omits 5, so a valid topological ordering (e.g. 1â†’3â†’2â†’5â†’4â†’6) runs Batch 2 before Batch 5; the verify `go test ./internal/wiki/` then fails to compile (`atomicWrite` undefined).
**Fix:** Either change Batch 2's `depends-on` to `[1, 3, 5]`, or move `atomicWrite` into Batch 1 (it has no deps beyond stdlib and is already required by Batch 2's save path).

### [BLOCKING] Card 5 (store.go CRUD) missing layer.go in Context
**Location:** Batch 2 Card 5
**Issue:** `ListTasksBrief()` calls `computeLayers` from `layer.go` and uses `TaskWithLayer`, but `layer.go` is absent from Card 5's `Context:` field. The implementer cannot know `computeLayers`'s signature or `TaskWithLayer`'s definition without it.
**Fix:** Add `internal/wiki/layer.go` to Card 5's `Context:` list.

### [NIT] lock.go context lists task.go unnecessarily
**Location:** Batch 5 Card 13
**Issue:** `lock.go` wraps only `github.com/gofrs/flock`; `task.go` contributes nothing to its implementation.
**Fix:** Remove `internal/wiki/task.go` from Card 13's `Context:`.

### [NIT] BriefTask JSON tags for Layer and HasProposal not named explicitly
**Location:** Batch 2 Card 5
**Issue:** The spec says "matching JSON tags" for `Layer string` and `HasProposal bool` without stating them; the implementer must infer `layer` and `has_proposal`.
**Fix:** Spell the tags out: `json:"layer"` and `json:"has_proposal"`.
