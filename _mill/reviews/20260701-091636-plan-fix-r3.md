## Fixed

- **[BLOCKING] `--undo` step 4 breaks the "clean no-op on never-paired host" goal**
  (`04-initcli-undo.md`, Card 8 step 4 / Card 10) — verified
  `weftengine.Commit`'s internal `ensureLockDir` (`internal/weftengine/sync.go`) calls
  `os.MkdirAll(weftPath + "/.weft", ...)` unconditionally, which would silently create
  a stray `<slug>-weft/.weft/` directory tree and then fail on the subsequent `git add`
  when `l.WeftWorktree()` doesn't exist at all (a truly-unpaired host — no `warp add`
  ever run). Added a `os.Stat(l.WeftWorktree())` existence guard to Card 8 step 4:
  when the weft worktree itself doesn't exist, skip `os.RemoveAll`, `Commit`, and
  `Push` entirely and report `weft_content: "not_present"`, restoring the clean no-op
  the "no separate pre-gate" Shared Decision requires. Added
  `TestRunInit_Undo_NoWeftPairing` to Card 10, mirroring `initcli_test.go`'s existing
  `TestRunInit_NoPairing` bare-repo fixture, asserting no `<slug>-weft` directory is
  created as a side effect.
