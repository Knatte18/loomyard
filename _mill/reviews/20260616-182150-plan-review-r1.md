## Findings

### [NIT] WriteJSON spec swallows MarshalIndent error
**Location:** Batch 4, Card 8 (and `## Shared Decisions → internal/state API`)
**Issue:** Spec reads `data, err := json.MarshalIndent(...); return fsx.AtomicWriteBytes(path, data)` — the marshal `err` is assigned but never checked, so a marshal failure would write garbage/nil. The muxpoc.SaveState model referenced in the same card does check and wrap this error.
**Fix:** Insert `if err != nil { return fmt.Errorf("marshal: %w", err) }` between the marshal and the atomic write in both the Shared Decisions block and Card 8 Requirements.

### [NIT] fsx_test.go import list omits the fsx package
**Location:** Batch 1, Card 2
**Issue:** Requirements says "Use only stdlib `testing`/`os`/`path/filepath`/`strings`" — but the external `package fsx_test` must also import `github.com/Knatte18/loomyard/internal/fsx` to call `fsx.PathGuard` / `fsx.AtomicWrite` / `fsx.AtomicWriteBytes`.
**Fix:** Add the fsx package import to the listed imports, or rephrase as "stdlib plus the package-under-test."
